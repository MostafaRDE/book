# Coffee Shop Debugging

The best debugging happens away from the desk — not because the coffee is magic, but because **changing context changes cognition**.

## My unofficial protocol

1. Reproduce the bug once, write down symptoms
2. Walk to coffee
3. Explain the problem to an imaginary patient friend
4. Return with either a fix or acceptance

Flat books like this one are for fragments. Not every thought needs a part, a preface, or a conclusion. Sometimes a chapter is just a note that deserved to be saved.
